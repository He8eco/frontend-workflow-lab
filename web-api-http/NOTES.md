## Substage 1 — Block 0
- что возвращает fetch - fetch возвращает Promise
- что такое Response - это объект, который описывает HTTP-ответ
- чем response отличается от data - response это объект, который содержит в себе ответ целиком, заголовки, статус, флаг и само тело ответа, а в data через response.json или response.text читаем само тело ответа и выводим
- зачем нужны status и ok - status нужен чтоб видеть статус ответа конкретно, 200, 201, 400, 401 и так далее, а ok выдаёт булевое значение, либо true либо false, true означает что ответ получен, а false означает что возникла ошибка.
- почему json() и text() — это отдельный шаг. Потому что сначала получает объект Response, а потом уже только через json() и text() получаем сам ответ, который нам необходим
- почему часто нужно два await - потому что нужно не только подождать чтоб получить ответ, но ещё и подождать чтоб распарсить ответ.

## Substage 1 — Block 1
- почему 404 не обязан попадать в catch - потому что при статусе 404 запрос был успешно отправлен на сервер и с сервера ответ тоже вернулся со статусом, а в catch ошибки попадают в том случае, если по той или иной причине запрос был прерван, к примеру на сервер или с сервера, из-за сбоя сети или ещё чего-нибудь
- зачем нужен response.ok - он указывает успешный статус в диапазоне 200-299, в котором ответ возвращается, а соответственно с помощью него удобно фильтровать плохой HTTP ответ от хорошего
- чем плохой HTTP-ответ отличается от сетевой ошибки - плохой HTTP-ответ это когда запрос завершился, но с неуспешным статусом, а соответственно ответ остаётся в try. А сетевая ошибка это когда уже произошёл сбой в отправке запроса, при возврате ответа от браузера и так далее, то есть когда запрос не смог завершиться
- почему try/catch и проверка ok — это не одно и то же. Потому что при плохом HTTP-ответе try не передаёт ошибку в catch, в то время как проверка ok указывает плохой HTTP-ответ как false, то есть позволяет обнаруживать плохие HTTP-ответы.

## Substage 1 — Block 2
- Базовая практика `fetch` привязана к UI-стенду
- По кнопке Load запускается запрос и обновляется интерфейс
- Успешный ответ показывает статус `success` и выводит данные в Result
- Неуспешный HTTP-ответ показывает статус `error` и текст ошибки
- Лог помогает видеть последовательность действий запроса
- `response.ok` используется для ручной проверки HTTP-статуса
